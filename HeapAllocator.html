Heap Allocator (CS107 Final Assignment)

As part of Stanford’s CS107 “Computer Organizations and Systems” course, I implemented two heap allocator designs in C:

  • An implicit free‐list allocator (unstable list weaving through heap blocks).
  • An explicit free‐list allocator using a doubly-linked free block list with coalescing for improved efficiency.

The project reinforced my skills in pointer manipulation, debugging, profiling with Valgrind, and designing data structures under low-level constraints. It was evaluated using a test_harness framework that exercised my implementations across simulated allocation workloads

Key implementation highlights:

  • Accurate handling of dynamic calls: malloc, free, and realloc.
  • Block headers with size and allocation status using least significant bits per spec.
  • Efficient memory reuse via explicit free‑list, enabling O(1) coalescing on free.
  • Profiling and performance tuning using Valgrind and callgrind.

Note: All source code is restricted by academic policy. Code is available upon request.
